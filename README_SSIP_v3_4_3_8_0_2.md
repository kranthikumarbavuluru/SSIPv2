# SSIP v3.4.3.8.0.2 — MeitY Review Compression

This phase solves the practical Admin workload problem created by broad
official-source discovery.

The v3.4.3.8.0.1 purification retains complete evidence, but the Admin
workspace still exposes dozens of individual records. v3.4.3.8.0.2 converts
those rows into three governed lanes.

## Lane 1 — Automatically resolved

No routine Admin action is required for:

- Page Not Found, Access Denied and generic navigation groups;
- repeated portal/error records;
- brochures, guidelines, approval orders, result documents and other
  supporting evidence;
- conferences, summits, expos, delegations, messages and similar
  non-catalogue evidence.

Every child record remains visible in the audit ledger and can be inspected.

## Lane 2 — Batch confirmation

One family-level decision is presented for:

- an existing canonical programme family;
- a group of calls or challenges linked to one family;
- historical result/event evidence;
- grouped low-risk identity classifications.

A bundle may permit one action for all children, or require selection of
specific child records.

## Lane 3 — Deep review

Individual review is preserved only for:

- possible current or upcoming calls;
- official application routes or deadlines;
- possible new permanent programme families;
- unresolved programme/call role conflicts;
- high-risk parent or identity conflicts.

## Governed workload limit

The generated Admin queue may contain no more than 20 decision bundles.
Low-risk bundles are compressed when required, while current calls and
possible new programmes remain protected as individual review items.

The exact count is evidence-dependent.

## Workspace

The family-level workspace runs on port 8508 and provides:

- workload-reduction metrics;
- priority and lane filters;
- family-level decision bundles;
- expandable child evidence;
- session-only Admin decisions;
- downloadable decision worksheet;
- grouped automatic-resolution audit;
- complete child ledger.

Session decisions are not written to the operational database. The exported
worksheet is intended for a later governed decision-import phase.

## Governance

- No database write.
- No publication.
- No automatic OPEN status.
- No Apply action.
- No source-evidence deletion.
- All source rows and consolidated evidence weights reconcile exactly.
- Automatic resolutions are reversible and auditable.
