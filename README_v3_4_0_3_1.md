# SSIP v3.4.0.3.1 — DST Category Discovery Gap Consolidation

## Purpose

This phase consolidates the `CATEGORY_INDEX_DISCOVERY_GAP` rows generated by
v3.4.0.3. It converts repeated category-link occurrences into one unique target
record, compares each target with the 33 provisional DST entities, excludes
calls and supporting pages, and produces a small review set of credible missing
schemes/programmes.

It does **not** crawl the web, modify prior outputs, create canonical identities,
or lock scheme/programme names.

- `POSSIBLE_NEW_PROGRAMME`

- `SUPPORTING_PAGE`

- `NON_SCHEME_PAGE`
- `BROKEN_OR_MISSING_TARGET`

## Approval gate

The phase is ready for v3.4.0.4 when both values are `true`:

```json
{
  "gap_validation_passed": true,
  "ready_for_v3_4_0_4": true
}
```

The strict command exits with code `3` if integrity validation fails, but still
writes the outputs for inspection.
